# Real-World Impact

CCR can represent real-world operation candidates as finite TRC traces, resource
ledgers, preflight reports, provider plans, dispatch reports, observations,
incidents, and residual repair tasks. It does not perform uncontrolled physical
actuation.

The profile ladder is:

- development: dry-run only;
- research: simulation or shadow only;
- controlled: fixture provider or explicitly local test provider;
- production: operator-approved provider dispatch with strict preflight and an
  observation verifier;
- adversarial: unknowns block by default.

Observation evidence is not physical outcome proof. Missing verifier acceptance,
rollback confirmation, hazard follow-up, or outcome matching creates residuals
and repair tasks.

Search terms: Typed Reality Compilation, TRC, operation profile, physical
provider, rollback escrow, incident ledger, observation verifier.

## v1.5 P2 Safe Review

First-time agents should use operation replay and observation verification only
to inspect local JSON evidence. Replay is not dispatch, verification is not physical outcome proof, and static workbench pages are presentation only. Use
the residual market to rank repair work; it does not waive residuals. Use
provider registry validation as static metadata review; registry metadata is
not authority.
